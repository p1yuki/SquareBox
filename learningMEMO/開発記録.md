## はじめに

こんにちは！このページは「SquareBox」というWebアプリを、**非エンジニアの開発初心者**が実際に作り、GitHubとGitHub Pagesで公開するまでの全記録です。

---

## どんなアプリを作ったの？

**SquareBox**は、写真をクロップせずに正方形に整え、InstagramなどのSNS投稿に最適な画像をローカルで簡単に生成できるWebアプリです。

- 写真をアップロード → 背景色やグラデーション・画像背景を選択 → 正方形に変換 → ダウンロード
- 複数画像の一括変換・ダウンロードもOK
- PWA対応でスマホやPCのホーム画面からアプリのように使える

---

## なぜこの技術・プラットフォームを選んだの？

### 1. **React + Vite + TypeScript**
- **React**：Webアプリを部品（コンポーネント）ごとに作れる人気のフレームワーク。初心者でも「見た目」と「動き」を分けて考えやすい。
- **Vite**：開発が速く、ビルドも一瞬。初心者でも「npm run dev」で即プレビューできる。
- **TypeScript**：JavaScriptに「型」を加えた言語。間違いを事前に防げるので、初心者にも安心。

### 2. **PWA（Progressive Web App）**
- **理由**：スマホやPCのホーム画面に追加でき、オフラインでも使える。アプリストア不要で配布も簡単。
- **メリット**：インストール不要・オフライン対応・通知も可能
- **デメリット**：一部のiOS機能制限、ストア配信アプリほどの自由度はない

### 3. **GitHub & GitHub Pages**
- **GitHub**：ソースコードや履歴をクラウドで管理できるサービス。無料で使えて、世界中の開発者が利用。
- **GitHub Pages**：GitHub上のファイルをそのままWebサイトとして公開できる超便利な機能。
- **バージョン管理**：過去の状態に戻せる「タイムマシン」みたいなもの。初心者でも安心して実験・修正できる。

## フォルダ構成と各ファイルの役割

```
SquareBox/
├─ assets/           # 画像やビルド後の静的ファイル
├─ dist/             # 公開用ビルドファイル（npm run buildで生成）
├─ learningMEMO/     # 学びや開発記録（このファイルもここ！）
├─ node_modules/     # ライブラリ群（npm installで自動生成）
├─ public/           # 公開用の静的ファイル（favicon等）
├─ src/              # アプリ本体のソースコード
│    └─ square-image-app.tsx  # メインのReactコンポーネント
├─ .gitignore        # Gitで管理しないファイル一覧
├─ index.html        # アプリのHTMLエントリーポイント
├─ package.json      # プロジェクト設定・依存ライブラリ一覧
├─ tailwind.config.js# CSSフレームワーク設定
├─ tsconfig.json     # TypeScriptの設定
├─ vite.config.ts    # Viteのビルド・公開設定
└─ ...
```

- **src/square-image-app.tsx**：アプリのメインロジック。画像アップロード・変換・ダウンロード・UI制御など全部ここ！
- **public/**：faviconやPWA用のマニフェストなど、公開に必要な静的ファイル
- **dist/**：ビルド後の「Web公開用」ファイル。これがGitHub Pagesで公開される
- **package.json**：npmコマンドや依存ライブラリの管理ファイル
- **learningMEMO/**：学びや開発の記録を残すフォルダ

## 開発の流れ（初心者向けステップバイステップ）

1. **プロジェクト作成・初期設定**
   - npmでReact+Vite+TypeScriptのプロジェクトを作成
   - 必要なライブラリ（react-easy-cropなど）をnpmで追加

2. **アプリの実装**
   - 画像アップロード・プレビュー・背景色/グラデ/画像背景選択・クロップ・ダウンロード機能を実装
   - UIはTailwindCSSで簡単＆おしゃれに

3. **PWA対応**
   - manifest.jsonやvite-plugin-pwaでPWA化
   - スマホやPCのホーム画面から起動できるように

4. **バージョン管理（Git/GitHub）**
   - 変更ごとに「コミット」して履歴を残す
   - 「タグ」でv1.0/v2.0などバージョンを明示
   - つまずきポイント：どのブランチで作業しているか分からなくなる→`git branch`や`git status`で確認！

5. **ビルド＆GitHub Pages公開**
   - `npm run build`でdistフォルダ生成
   - `npm run deploy`でgh-pagesブランチにアップロード→Web公開
   - つまずきポイント：ビルドエラーは`npm install`で解決することが多い
   - 公開後は「スーパーリロード（Ctrl+F5）」で最新UIを確認

6. **バージョン管理の魅力**
   - タグ（v1.0, v2.0）で過去の状態にいつでも戻せる
   - main/gh-pagesブランチの使い分けも学べた！

## PWAを選んだ理由・メリット・デメリット

### メリット
- インストール不要で、スマホやPCのホーム画面からすぐ起動できる
- オフラインでも使える
- アプリストア申請不要、配布が超簡単

### デメリット
- iOSでは一部機能制限あり（通知やファイル保存など）
- ストアアプリほどの自由度はない

## GitHub & GitHub Pages入門（初心者のつまずきポイント付き）

### GitHubとは？
- ソースコードやファイルの「履歴」をクラウドで管理できるサービス
- どんな変更も「コミット」で記録、過去の状態に戻せる
- 「タグ」や「ブランチ」でバージョン管理も簡単

### GitHub Pagesとは？
- GitHub上のファイルをそのままWebサイトとして公開できる超便利な機能
- 静的サイト（HTML/CSS/JS）なら無料で公開OK
- ブランチやフォルダを選んで公開元を切り替えられる

### つまずきポイントと解決法
- **どのブランチで作業しているか分からない** → `git branch`や`git status`で確認
- **ビルドエラー** → まず`npm install`で依存パッケージを揃える
- **公開後にUIが変わらない** → スーパーリロード（Ctrl+F5）でキャッシュをクリア
- **タグやバージョンの使い方が分からない** → `git tag`でバージョンを明示、`git log`で履歴を確認

### バージョン管理の魅力
- 失敗してもすぐに「過去の状態」に戻せる安心感
- いろんなバージョンをタグで管理できる
- チーム開発や実験にも最適
